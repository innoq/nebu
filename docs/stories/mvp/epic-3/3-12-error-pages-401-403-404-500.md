# Story 3.12: Error Pages (401, 403, 404, 500)

Status: done

## Story

As an operator,
I want clear, on-brand error pages for all HTTP error conditions in the Admin UI,
so that I understand what went wrong without seeing a raw browser error or blank page.

## Acceptance Criteria

1. `gateway/internal/admin/templates/errors/` contains: `401.html`, `403.html`, `404.html`, `500.html`
2. Each template extends the base layout and renders within the same shell (topbar + sidebar)
3. 401 page: heading "Authentication Required", message "Your session has expired or you are not logged in.", link **Log in again** → `/admin/login`
4. 403 page: heading "Access Denied", message "You do not have permission to access this page.", link **Back to Dashboard** → `/admin/dashboard`
5. 404 page: heading "Page Not Found", message "The page you requested does not exist.", link **Back to Dashboard** → `/admin/dashboard`
6. 500 page: heading "Internal Server Error", message "Something went wrong. Please try again or contact your system administrator.", no stack trace exposed
7. `gateway/internal/admin/errors.go` exports helpers `Error401(w, r, h)`, `Error403(w, r, h)`, `Error404(w, r, h)`, `Error500(w, r, h)` that set the correct HTTP status code and render the corresponding template
8. The `/admin/*` router in `main.go` registers a `NotFound` handler that calls `Error404`
9. A unit test for each helper asserts correct HTTP status code and that the response body contains the expected heading

## Dev Notes

### Template Architecture: How `TemplateHandler` and `render()` Work

**CRITICAL: Read this before writing any template code.**

`TemplateHandler` (in `gateway/internal/admin/handler.go`) uses per-page isolated template sets:

- Templates are discovered by walking `templates/` recursively via `fs.WalkDir`.
- Layout templates (`templates/layouts/*.html`) are shared.
- Each page template (any `.html` NOT under `templates/layouts/`) gets its own compiled `*template.Template` set containing: layouts + that single page file.
- The map key is the file's `path.Base` **without extension**, e.g., `templates/errors/401.html` → key `"401"`.

**PROBLEM**: If two page templates have the same base filename (e.g., `templates/errors/401.html` and hypothetically `templates/401.html`), only the last one wins in the `pageTmpls` map. The current codebase has no conflicts, but ensure the error template filenames remain distinct.

**Key lookup**: `h.render(w, "401", data)` → looks up `pageTmpls["401"]` → executes template named `"base"` with data. This means:
- Template file name WITHOUT extension = render key
- Use `h.render(w, "401", data)` NOT `h.render(w, "errors/401", data)`

The `WalkDir` in `NewTemplateHandler` **already recursively discovers** `templates/errors/401.html` because it walks all of `templates/`. No code changes to `handler.go` are needed.

### Error Helper Signature Pattern

The AC specifies `Error401(w, r, h)`. Use the `*TemplateHandler` as the third parameter (not a method on it) to match the package-level function pattern used for `BootstrapGuard` and `SessionGuard`:

```go
// gateway/internal/admin/errors.go
package admin

import "net/http"

// Error401 writes HTTP 401 and renders the "Authentication Required" error page.
func Error401(w http.ResponseWriter, r *http.Request, h *TemplateHandler) {
    w.WriteHeader(http.StatusUnauthorized)
    h.render(w, "401", PageData{})
}

// Error403 writes HTTP 403 and renders the "Access Denied" error page.
func Error403(w http.ResponseWriter, r *http.Request, h *TemplateHandler) {
    w.WriteHeader(http.StatusForbidden)
    h.render(w, "403", PageData{})
}

// Error404 writes HTTP 404 and renders the "Page Not Found" error page.
func Error404(w http.ResponseWriter, r *http.Request, h *TemplateHandler) {
    w.WriteHeader(http.StatusNotFound)
    h.render(w, "404", PageData{})
}

// Error500 writes HTTP 500 and renders the "Internal Server Error" error page.
func Error500(w http.ResponseWriter, r *http.Request, h *TemplateHandler) {
    w.WriteHeader(http.StatusInternalServerError)
    h.render(w, "500", PageData{})
}
```

**CRITICAL `WriteHeader` ordering**: `w.WriteHeader(statusCode)` MUST be called BEFORE `h.render(...)`. The `render()` function calls `w.Header().Set("Content-Type", ...)` which is allowed after `WriteHeader`, but calling `WriteHeader` after writing the body header would silently be ignored. Follow this exact ordering.

**DO NOT** set `Content-Type` manually in the error helpers — `h.render()` already sets `Content-Type: text/html; charset=utf-8`.

### Template Structure to Follow

All templates in this codebase use this exact pattern (see `login.html`, `bootstrap.html`):

```html
{{ template "base" . }}
{{ define "title" }}PAGE TITLE — Nebu Admin{{ end }}
{{ define "content" }}
<!-- page-specific HTML here -->
{{ end }}

{{ define "scripts" }}{{ end }}
```

The `{{ define "scripts" }}{{ end }}` block must be defined even if empty, because the base layout defines `{{ block "scripts" . }}{{ end }}` and the template set needs a concrete definition to avoid "template: base:XX: missing definition" errors.

### `NotFound` Handler in `main.go`

Go's `net/http` ServeMux (1.22+) with method-prefixed patterns selects the most specific route first. The catch-all pattern `"GET /admin/"` will only match requests to `/admin/` and below that do NOT match any more specific registered pattern.

Add this after all other `/admin/*` route registrations in `main.go`:

```go
// Catch-all 404 for unmatched /admin/* paths
mux.HandleFunc("GET /admin/", func(w http.ResponseWriter, r *http.Request) {
    admin.Error404(w, r, tmplHandler)
})
```

Note: `tmplHandler` is already in scope (created earlier in `main.go`). This does NOT wrap with `sessionGuard` or `guard` because the 404 page must be reachable even for unauthenticated requests hitting unknown paths — attempting to access a guarded route that doesn't exist would first fail the guard and redirect to login, which is fine.

### Test Pattern: Using `NewTemplateHandler()` Directly

All tests in the `admin` package use `package admin` (NOT `package admin_test`). Test files can call `NewTemplateHandler()` directly as demonstrated in `handler_test.go`.

The test pattern for error helpers:

```go
package admin

import (
    "net/http"
    "net/http/httptest"
    "strings"
    "testing"
)

func TestError401_StatusAndHeading(t *testing.T) {
    h, err := NewTemplateHandler()
    if err != nil {
        t.Fatalf("NewTemplateHandler: %v", err)
    }
    w := httptest.NewRecorder()
    r := httptest.NewRequest("GET", "/admin/dashboard", nil)
    Error401(w, r, h)
    if w.Code != http.StatusUnauthorized {
        t.Errorf("status: got %d, want %d", w.Code, http.StatusUnauthorized)
    }
    if ct := w.Header().Get("Content-Type"); ct != "text/html; charset=utf-8" {
        t.Errorf("Content-Type: got %q, want %q", ct, "text/html; charset=utf-8")
    }
    if !strings.Contains(w.Body.String(), "Authentication Required") {
        t.Error("expected body to contain 'Authentication Required'")
    }
}
```

For `TestError500_NoStackTrace`:
```go
func TestError500_NoStackTrace(t *testing.T) {
    h, err := NewTemplateHandler()
    if err != nil {
        t.Fatalf("NewTemplateHandler: %v", err)
    }
    w := httptest.NewRecorder()
    r := httptest.NewRequest("GET", "/admin/dashboard", nil)
    Error500(w, r, h)
    body := w.Body.String()
    for _, forbidden := range []string{"goroutine", "panic", "runtime/"} {
        if strings.Contains(body, forbidden) {
            t.Errorf("error page must not contain %q", forbidden)
        }
    }
}
```

### Anti-Patterns to Avoid

- **DO NOT** call `h.render(w, "errors/401", data)` — the key is `"401"` (base filename without extension), NOT the full path. The `pageTmpls` map keyed by `path.Base(file)` with extension stripped.
- **DO NOT** call `w.WriteHeader()` AFTER writing any body content — the status code will be silently ignored.
- **DO NOT** expose `err.Error()` strings in 500 responses. Log errors via `slog.Error(...)` at the call site before calling `Error500`.
- **DO NOT** add `{{ define "scripts" }}{{ end }}` as an optional block — it MUST be defined to avoid template execution errors.
- **DO NOT** create a new `ErrorPageData` struct — `PageData{}` with zero values is correct (no nav item highlighted).
- **DO NOT** modify `handler.go` to add special handling for the `errors/` subdirectory — the existing `WalkDir` recursion already handles it.
- **DO NOT** use `http.Error(w, ...)` — this produces plain-text responses, not on-brand HTML error pages.
