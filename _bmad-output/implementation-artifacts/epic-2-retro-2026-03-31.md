## Challenges

### 1. OTP Crypto API Variance (Stories 2-8, 2-9, 2-10, 2-11)

**What happened:** Each OTP crypto primitive has a different API shape: Ed25519 sign uses a list `[key, :ed25519]`, X25519 ECDH uses a 4th argument for the curve, AES-GCM uses a 7-arg function. OTP 28 changed `crypto_one_time_aead` to return `:error` instead of raising, requiring a `case + rescue` dual pattern for compatibility.

**Impact:** Required empirical validation in each story. Patterns are now documented in story Dev Notes.

**Resolution:** Established OTP Crypto pattern library in story records. Context7 MCP will be used in future to load current OTP docs before implementation.

### 2. Dex ROPC Surprise (Story 2-21)

**What happened:** Architecture planning assumed ROPC (Resource Owner Password Credentials) could be used for Gherkin tests. Dex v2.41 does not support ROPC. Story 2-21 had to implement full Authorization Code + PKCE programmatic navigation.

**Root cause:** External tool API assumptions not verified during planning. A 5-minute Dex docs check would have revealed this.

**Resolution:** New convention — verify grant type support before implementing auth-related stories. Context7 MCP for current docs on all external dependencies.

**Note:** The outcome was positive — the test now validates the real client flow, not a simplified shortcut.

### 3. `mapRole` Duplication (Stories 2-5, 2-6, 2-18)

**What happened:** Identical switch statement implemented three times: `middleware.mapRole`, `admin.mapAdminRole`, `matrix.mapSystemRole`.

**Impact:** Story 6-3 (role overrides from DB) would need to update all three simultaneously. Missing one guarantees a bug.

**Resolution:** Cleanup Task C2 — extract to `gateway/internal/auth/roles.go` shared function before Epic 3.

### 4. In-Memory Denylist Architecture (Story 2-19)

**What happened:** Logout denylist implemented as `sync.Map` with SHA-256(token) + lazy TTL expiry in Go gateway. Intentional MVP tradeoff — no shared state across instances, disappears on restart.

**Root cause:** Avoidance of per-request DB lookup for performance. However, `sessions` table already exists and Elixir already invalidates sessions on logout. The `ValidateToken` gRPC path can perform session-active check as part of existing validation.

**Resolution:** Cleanup Task C1 — replace in-memory denylist with DB-backed session check via `ValidateToken` gRPC call. Removes split-brain between in-memory and PostgreSQL.

## Key Insights

1. **Dex Authorization Code Flow is the new standard.** PKCE is mandatory for this flow. All future Gherkin tests involving OIDC must use Authorization Code + PKCE. Document this as a Nebu testing convention.

2. **The DB is the denylist.** An existing `sessions` table + existing `ValidateToken` gRPC call makes a separate in-memory denylist architecturally redundant. Use the infrastructure you have.

3. **Context7 MCP eliminates empirical API discovery.** OTP `:crypto` variance, Dex ROPC absence, `go-oidc` behavior — all discoverable in minutes with current docs. Load docs before implementation, not after.

4. **Playwright MCP for HTML/UI E2E tests.** Godog + `net/http` is correct for REST API and gRPC-level tests. Browser-level tests (forms, clicks, redirects) need a real browser. Playwright is the tool.

5. **Small stories remain the right call.** Epic 2 introduced cryptographic operations, OIDC JWT validation, and Elixir session management — all domains where catching errors at story boundaries matters. 21 small stories, 0 incidents.
