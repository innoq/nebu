## Dev Notes

### Critical: Remove Existing Placeholder Files First

The root `main.go` and `go.mod` are GoLand IDE auto-generated placeholders with **no project relevance**:
- `go.mod` has `module open-chat` — wrong module path, wrong location
- `main.go` is a "Hello World" toy file generated by the IDE

**Both must be deleted.** The project's Go code lives under `gateway/` and `media/`, each with their own `go.mod`.

### Go Module Layout

Two separate Go modules — NOT a Go workspace:

```
gateway/go.mod    → module github.com/nebu/nebu
media/go.mod      → module github.com/nebu/nebu/media
```

Later, `proto/gen/go/` will be a third Go module imported via `replace` directive in both `gateway/go.mod` and `media/go.mod`. Do NOT add this replace directive in Story 1.1 — it will be added in Story 1.6 when proto stubs are generated.

### Config Struct Pattern

`gateway/internal/config/config.go` must define all NEBU_ env vars. The `NEBU_INTERNAL_SECRET_FILE` pattern is critical — read the file path, then read the file content:

```go
package config

import "os"

type Config struct {
    CoreGRPCAddr       string // NEBU_CORE_GRPC_ADDR   (default: "core:9000")
    DBURL              string // NEBU_DB_URL
    OIDCIssuer         string // NEBU_OIDC_ISSUER
    InternalSecretFile string // NEBU_INTERNAL_SECRET_FILE (path to mounted secret)
    ServerName         string // NEBU_SERVER_NAME
}

func Load() Config {
    return Config{
        CoreGRPCAddr:       getEnvOrDefault("NEBU_CORE_GRPC_ADDR", "core:9000"),
        DBURL:              os.Getenv("NEBU_DB_URL"),
        OIDCIssuer:         os.Getenv("NEBU_OIDC_ISSUER"),
        InternalSecretFile: os.Getenv("NEBU_INTERNAL_SECRET_FILE"),
        ServerName:         os.Getenv("NEBU_SERVER_NAME"),
    }
}
```

**NEVER** read the secret value directly from an env var (`os.Getenv("NEBU_INTERNAL_SECRET")`). Always read the file path and then read the file contents.

### main.go Placeholder Pattern

Use `log/slog` (stdlib since Go 1.21) for structured logging — this project never uses `fmt.Println` or `log` (standard library non-structured logger):

```go
package main

import (
    "log/slog"
    "os"
)

func main() {
    slog.Info("Nebu Gateway starting")
    os.Exit(0)
}
```

### Makefile: Docker-Only Build System

All build commands run inside Docker containers. **No local Go, Elixir, or buf installation is required.** This is a hard requirement.

```makefile
DOCKER_GO    = docker run --rm -v $(PWD):/workspace -w /workspace golang:1.23-alpine
DOCKER_ELIXIR = docker run --rm -v $(PWD):/workspace -w /workspace elixir:1.18-alpine
DOCKER_BUF   = docker run --rm -v $(PWD):/workspace -w /workspace bufbuild/buf
```

For Story 1.1, all targets are stubs that `echo` what they will do — the actual implementations come in later stories. The `make -n <target>` test verifies that the target is defined and the recipe syntax is valid.

### Gateway Dockerfile — Multi-Stage (gcr.io/distroless/static)

Use `gcr.io/distroless/static` as the runtime image (NOT alpine), as specified in the architecture. This is smallest possible runtime for a static Go binary:

```dockerfile
FROM golang:1.23-alpine AS builder
WORKDIR /workspace
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN go build -o /gateway ./cmd/gateway

FROM gcr.io/distroless/static AS runtime
COPY --from=builder /gateway /gateway
EXPOSE 8008 8448
ENTRYPOINT ["/gateway"]
```

| Exported types/funcs | PascalCase | `Config`, `Load()` |
| JSON struct tags | snake_case | `json:"room_id"` |
| Env var names | `NEBU_{COMPONENT}_{KEY}` | `NEBU_DB_URL`, `NEBU_OIDC_ISSUER` |
