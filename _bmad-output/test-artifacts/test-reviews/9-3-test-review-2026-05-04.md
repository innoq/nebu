#### M-1 — Playwright AC3: Ambiguous "Save" button selector (Strict Mode Violation)

**File:** `e2e/tests/features/admin/rooms-api-integration.spec.ts:244`  
**Selector:** `page.locator('section[role="region"] button[type="submit"]:has-text("Save")')`

**Problem:**  
The `detail_panel` (`section[role="region"]`) contains **two** `button[type="submit"]` elements with text "Save":

1. `rooms.html:105` — Settings form Save button (visible by default)
2. `inline_edit.html:19` — Inline edit form Save button (hidden via CSS `display:none` on parent `span.inline-edit-form`)

Playwright's strict-mode enforces single-element resolution for `.click()` calls. The selector matches **both** elements in the DOM (CSS `display:none` does not exclude elements from Playwright's strict-mode count). This will throw:

```
Error: strict mode violation: locator('section[role="region"] button[type="submit"]:has-text("Save")')
resolved to 2 elements
```

**Recommended Fix:**  
Scope the selector to the settings form by matching the containing `<form>` with `action` attribute:

```typescript
// Before (broken):
await page.locator('section[role="region"] button[type="submit"]:has-text("Save")').click();

// After (fixed):
await page.locator('form[action*="/settings"] button[type="submit"]').click();
```

Or use the unique form action:

```typescript
await page.locator(`form[action="/admin/rooms/${targetRoomId}/settings"] button[type="submit"]`).click();
```

#### MIN-1 — AC2 `afterEach` cleanup is best-effort only (CSRF bypass acknowledged but broken)

**File:** `e2e/tests/features/admin/rooms-api-integration.spec.ts:121–129`  
**Code:**
```typescript
await page.request.post(`/admin/rooms/${targetRoomId}/unarchive`);
```

**Problem:**  
`page.request.post` bypasses the browser's cookie/CSRF handling. The unarchive POST will receive HTTP 403 from the CSRF middleware. The comment acknowledges this ("NOTE (TEA MINOR-4): page.request.post bypasses CSRF middleware") but relies on the "next `beforeEach` finds the next active room."

The actual risk is test pollution: if only one room exists in the DB, the second AC2 test (`after archiving the room detail reflects...`) will attempt to archive an already-archived room. The archive button only renders when `room.Status == "active"` (template line 114: `{{ if eq .ActiveRoom.Status "active" }}`), so `page.locator('button:has-text("Archive room")').click()` would time out rather than immediately fail. The error message would be misleading.

**Severity:** MINOR (documented, test will fail with a timeout rather than a logic error; acceptable if ≥2 rooms exist in the bootstrap state)

**Recommendation:** Document the bootstrap prerequisite more explicitly: "prerequisite: at least 2 active rooms in DB" or use a before-test setup that creates rooms via the Matrix API. The story acceptance test already mentions "create a room via the Matrix API" as the AC1 sentinel check approach — extending this to provide cleanup state for AC2 would harden the suite.

#### MIN-3 — AC1 sentinel checks rely on stub name collision assumption

**File:** `e2e/tests/features/admin/rooms-api-integration.spec.ts:59,71`

**Problem:**  
The AC1 tests assert that "General" and "Engineering" are NOT visible. This assumes the production DB never has a room with those exact names. If an admin creates a room named "General", the test becomes a false negative (it passes incorrectly, then fails unexpectedly when the real DB grows).

This is a known weakness with sentinel-absence strategies (as documented in the Story 9.2 test-review: MINOR-2). The approach is acceptable for MVP but fragile long-term.

**Severity:** MINOR (acceptable for the current development phase; documented in the spec file header)

**Recommendation:** Consider using a UUID-based sentinel name in the future: create a room via the Matrix API with a generated name, assert it appears in the list, and assert that a specific UUID-keyed stub sentinel is absent. This is already hinted at in the story's acceptance test notes ("create a room via the Matrix API, then verify it appears").
