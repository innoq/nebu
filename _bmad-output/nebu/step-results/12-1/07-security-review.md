# Step 7: Security Review (Kassandra) — Story 12-1

**Verdict: 0 CRITICAL, 0 HIGH, 2 MEDIUM (advisory, dev-only)**

## Scope

Story 12-1 adds: MinIO service to docker-compose.yml, credential generation in make setup,
ADR-013, README MinIO section. `security_review: required` in story frontmatter.

## Findings

### MEDIUM-1: MinIO ports 9000/9001 published to host

- **File:** `docker-compose.yml`
- **Risk:** Developer running `make dev` on a shared network exposes MinIO to local network.
- **Mitigation:** Credentials are unique per developer (generated by `make setup` via `openssl rand`); no default "minioadmin" credentials used. Bucket ACL defaults to private (no public access).
- **Verdict:** Acceptable for local dev. Production operators must supply own config (per ADR-013/ADR-012). Deferred to Story 12.3 for production hardening.

### MEDIUM-2: Secret file permissions (umask 022 → mode 644)

- **File:** `Makefile` (setup target)
- **Risk:** `.secrets/minio_root_user` and `.secrets/minio_root_password` created without explicit `chmod 600`, inheriting umask. On default umask 022, world-readable.
- **Mitigation:** Consistent with existing `internal_secret` generation pattern (no chmod there either). Files are gitignored. Local dev machine only.
- **Verdict:** Advisory. Deferred to a hardening pass. Not blocking.

## Passed Checks

- No literal credentials in docker-compose.yml (MINIO_ROOT_USER_FILE pattern used correctly)
- createbuckets reads credentials from /run/secrets/ at runtime — no hardcoded values
- openssl rand -hex 16 generates 128-bit entropy (well above MinIO's 8-char minimum)
- No plaintext credential values in make setup output (only filenames echoed)
- Bucket nebu-media created with default private ACL (no --policy public)
- No new SQL, routes, or auth changes
