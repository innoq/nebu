## Challenges

### 1. Story 3-8: sync.Map Architecture — 3 Code Review Rounds

**What happened:** The initial Bootstrap draft storage used `sync.Map` (in-memory). Code review Round 1 correctly identified this as architecturally wrong: non-restart-resilient, multi-instance unsafe, state lost on gateway restart.

**Root cause:** No upfront acceptance test existed that required "Given a gateway restart, the bootstrap draft must survive." Without that constraint written down, the path-of-least-resistance implementation (in-memory) passed all existing unit tests.

**Resolution:** Refactored to PostgreSQL `bootstrap_draft` table. Three review rounds total. The correct solution was found — but late.

**Lesson:** An acceptance test expressing restart-resilience as a requirement would have driven the correct architecture from the start (ATDD).

### 2. Story 3-15: E2E Test Shortcuts — No Real Browser Flow

**What happened:** The Gherkin scenarios in Story 3-15 bypassed the actual UI by:
- Forging `admin_session` cookies directly via HMAC-SHA256
- Seeding `server_config` directly via PostgreSQL TRUNCATE + INSERT
- Never clicking through the Bootstrap Wizard in a real browser

**Root cause:** The Playwright MCP convention (P2) was established in the Epic 2 retro but not applied. The `bootstrap_wizard_test.go` unit tests cover form rendering but not the full browser-level click-through.

**Phil's insight:** No happy-path test existed for the entire epic. The correct approach is ATDD: define the happy-path Playwright scenario first, then implement against it.

**Resolution:** See Action Items P3 and the Epic 4 Playwright story (B2).

### 3. `extractFirstRoleClaim` Design — Deferred from Story 3-9

**What happened:** The role claim mapping only checks the first element of the claims array. If an OIDC provider returns `["viewer", "instance_admin"]`, only `"viewer"` is evaluated — the user never gets admin access. Additionally, `"instance_admin"` is hardcoded — enterprise deployments with custom group claims (e.g., `"corp_admin"`, `"nebu_admin"`, `"admin"`) cannot configure this.

**Root cause:** Single-value assumption in early OIDC implementation (Story 2-5). Not caught because tests only used single-value claim arrays.

**Resolution:** C3 (see Action Items). Full Bootstrap Wizard redesign with configurable `admin_group_claim` and "Connect with OIDC" flow that captures real claims.

## Key Insights

1. **ATDD prevents architecture mistakes.** Story 3-8's three review rounds would have been one if an acceptance test had expressed the restart-resilience requirement upfront. The spec drives the design.

2. **Cookie forging is not E2E testing.** Forging session cookies validates the code path but never validates the user experience. The Bootstrap Wizard multi-step form was never tested end-to-end in a real browser during this entire epic.

3. **Configurable > hardcoded for enterprise.** Every organization has different OIDC group claim names. Hardcoding `"instance_admin"` is a deployment blocker for any real customer. The Bootstrap Wizard must expose this configuration.

4. **"Connect with OIDC" during bootstrap = validate config AND capture claims in one step.** Instead of a "Test Connection" stub + a separate claim-selection step after first login, the wizard uses a real Authorization Code + PKCE flow at Step 2. The callback handler detects bootstrap mode and shows discovered claims for mapping. One real flow replaces two stub interactions.

5. **Feature-file-first is the Epic 4 standard.** The Playwright happy-path scenario is written before Story 4-1 starts. It is the living definition of "done" for the admin UI.

## Patterns Established (for use in Epic 4+)

| Pattern | Location | Description |
|---|---|---|
| go:embed isolated templates | `gateway/internal/admin/handler.go` | `fs.WalkDir` recursive discovery; per-page `template.New()` to prevent `{{ define "content" }}` conflicts |
| Admin whitebox tests | `gateway/internal/admin/*_test.go` | `package admin` (not `package admin_test`); direct access to unexported helpers; `httptest.NewRecorder` |
| BootstrapGuard + SessionGuard | `gateway/internal/admin/middleware.go` | Path-exclusion lists; composition via `func(http.Handler) http.Handler`; context value storage with typed unexported keys |
| AES-256-GCM stdlib secrets | `gateway/internal/admin/crypto.go` | `sha256.Sum256(secret)` → 32-byte key; nonce prepended to ciphertext (hex-encoded in DB) |
| Bootstrap draft store | `gateway/internal/admin/bootstrap.go` | `BootstrapDraftStore` interface; `postgresBootstrapDraftStore` with upsert; `fakeBootstrapDraftStore` for tests |
| SSE handler pattern | `gateway/internal/admin/sse.go` | `r.Context()` for lifecycle; `http.Flusher` for wire format; `flusherRecorder` test helper for long-running handler tests |
| Vue 3 ESM self-hosted | `gateway/internal/admin/static/vendor/` | `vue.esm-browser.prod.js` embedded; `Cache-Control: immutable`; ES module import in metrics-widget.js |

**Motivation:** `extractFirstRoleClaim` silently drops multi-value claims. `"instance_admin"` is hardcoded and not operator-configurable. Enterprise deployments have custom group claim names. The "Test Connection" stub is replaced by a real OIDC flow that validates config AND surfaces claims in one step.

**New Bootstrap Wizard Flow (4 steps, down from 4+stub):**

| Step | What |
|---|---|
| Step 1 | Instance name |
| Step 2 | OIDC Config (Issuer, Client ID, Client Secret) → **"Connect with OIDC"** button → triggers real PKCE flow |
| Callback | Token arrives → page shows: "Discovered claims: `[...]` — select your admin claim" → saved to draft as `admin_group_claim` |
| Step 3 | Key generation (Ed25519 fingerprint display) |
| Step 4 | Finalize → operator already authenticated → direct redirect to Dashboard |

**Implementation scope:**
- `server_config`: new key `admin_group_claim` (default: `"instance_admin"`)
- `CallbackHandler`: detect bootstrap mode → render claim-selection page instead of redirecting
- `extractFirstRoleClaim` → `matchesAdminGroupClaim(claims []string, db) bool`: loads `admin_group_claim` from `server_config`, checks ALL elements of claims array
- Remove `POST /admin/bootstrap/test-oidc` endpoint (replaced by real PKCE flow)
- Remove or replace `bootstrap-done.html` (D3 resolved — use session data)
- Fix catch-all `GET /admin/` handler: DB error → 500 not 404 (D2 resolved)
