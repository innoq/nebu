# Code of Conduct

## Our commitment

We want Nebu to be a project where everyone — regardless of background, experience level, identity, or role — can participate and feel respected. Contributors, maintainers, issue authors, and anyone else interacting in Nebu's spaces all share this commitment.

## Expected behavior

- Be welcoming and patient, especially with newcomers.
- Assume good faith. Most misunderstandings are misunderstandings.
- Give and accept constructive feedback gracefully. Critique ideas, not people.
- Prefer public, searchable discussion (issues, PR comments) over private channels, so the next person can learn from it.
- Take responsibility for mistakes, apologize, and move on.

## Unacceptable behavior

- Personal attacks, insults, or dismissive language directed at other contributors.
- Discriminatory remarks or jokes based on someone's identity or background.
- Unwelcome personal attention, persistent unwanted contact, or intimidation.
- Publishing others' private information without explicit permission.
- Sustained disruption of discussions or project work.

## Scope

This Code of Conduct applies in all project spaces — the repository, issues, pull requests, discussions, and any communication channel linked from the project — as well as when representing Nebu in public.

## Reporting

If you see or experience behavior that violates this Code of Conduct, please report it to the maintainers at **conduct@nebu.chat** (or directly to the repository owner if that address is not yet live). Reports are handled confidentially.

When reporting, include:

- What happened and where (link to the issue, PR, or comment if applicable)
- Who was involved
- Any relevant context

Maintainers will review every report, respond within a reasonable time, and take action proportional to the situation. Possible outcomes include a private conversation, a public clarification, a temporary cooldown, or — for serious or repeated issues — a permanent ban from project spaces.

## Attribution

This Code of Conduct is inspired by and broadly aligned with the [Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/), which we recommend reading for a more detailed framing of the same principles.
