## Key Findings

### Open Matrix Media Interfaces

**Implementiert (Epic 12):**
- `POST /_matrix/media/v3/upload` ✅
- `GET /_matrix/media/v3/download/{serverName}/{mediaId}` ✅ (deprecated since v1.11)
- `GET /_matrix/media/v3/thumbnail/{serverName}/{mediaId}` ✅ (deprecated since v1.11)

**KRITISCH — Element Web (v1.11+ authenticated media, spec §Media Repository):**
1. `GET /_matrix/client/v1/media/config` — fehlt; Requires auth; Element Web ruft dies vor jedem Upload auf (max upload size)
2. `GET /_matrix/client/v1/media/download/{serverName}/{mediaId}` — fehlt; Requires auth; seit v1.12 SHOULD servers "freeze" unauthenticated downloads → neu hochgeladene Media nur über authentifizierte Endpoints erreichbar
3. `GET /_matrix/client/v1/media/thumbnail/{serverName}/{mediaId}` — fehlt; Requires auth; gleiche Begründung
4. `GET /_matrix/client/v1/media/download/{serverName}/{mediaId}/{fileName}` — fehlt; Requires auth

**MEDIUM — Deprecated aber noch genutzt:**
5. `GET /_matrix/media/v3/config` — fehlt; deprecated seit v1.11; ältere Clients nutzen es noch
6. `GET /_matrix/media/v3/download/{serverName}/{mediaId}/{fileName}` — fehlt; deprecated Variante mit Filename

**LOW — Async Upload Flow (nicht MVP):**
7. `POST /_matrix/media/v1/create` — fehlt; Pre-allocate MXC URI
8. `PUT /_matrix/media/v3/upload/{serverName}/{mediaId}` — fehlt; Upload to pre-allocated URI
9. `GET /_matrix/client/v1/media/preview_url` / `GET /_matrix/media/v3/preview_url` — fehlt; URL preview

### SHOULD Violations (download + thumbnail handlers)

Per CS API v1.18 §Media Repository:
- Fehlt: `Content-Security-Policy: sandbox; default-src 'none'; ...` — SHOULD
- Fehlt: `Cross-Origin-Resource-Policy: cross-origin` — SHOULD (added v1.4)

Both handler files (`media/internal/download/handler.go`, `media/internal/thumbnail/handler.go`) set `X-Content-Type-Options: nosniff` but not CSP or CORP headers.
