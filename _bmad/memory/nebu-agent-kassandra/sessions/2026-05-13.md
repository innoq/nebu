# Session 2026-05-13 (cont'd) — Epic 12 Definitive SEC Gate 2 (post Story 12.14)

## Trigger

Owner requested the definitive final epic-end SEC Gate 2 after Story 12.14 closed the last
shutdown gap (the `srv.Shutdown` non-wiring noted as a watch-flag after final4/12.13).

## Method

- `git diff 5020ec8..HEAD` (range unchanged) — focus on 12.14 delta `fbe51e7`.
- Verified the shutdown sequence end-to-end: signal → ctx cancel → `srv.Shutdown(30s)` → `pool.Close()` → exit 0.
- Adversarial questions answered explicitly:
  - Can shutdown phase be used for auth bypass? → No. `r.Context()` is NOT derived from the SIGTERM ctx (no `BaseContext` set). In-flight requests run full auth middleware on uncancelled request contexts.
  - Can rate limits be reset? → No. Cleanup goroutine exit doesn't clear the limiter `sync.Map`. New process = new map, but that's per-process semantics, not a 12.14 regression.
  - DB leak race? → No. Drain → then `pool.Close()`. Order is correct.
  - Signal-handler correctness? → Canonical `signal.NotifyContext` + `defer stop()`.
  - Goroutine / channel hygiene? → Buffered serverErr channel, three goroutines all join cleanly.

## Verdict

**CLEAN.** 0/0/0/3-LOW (advisories only, no follow-up needed). Epic 12 is complete for security.

## New pattern captured

Added to MEMORY: "Shutdown-phase auth bypass via cancelled request ctx" — a forward-looking
watch-flag. The current code is safe because Go's default `BaseContext` is `context.Background()`
and 12.14 did NOT override it. But if a future story tries to "speed up drain" by wiring
SIGTERM ctx as `BaseContext`, that would propagate cancellation into `r.Context()` and could
turn `oidcVerifier.Verify(ctx, ...)` into a `context.Canceled` error path. Depending on how that
error is mapped (401? bypass with "service shutting down, skip"?), it could open a window. Rule
recorded: never wire SIGTERM ctx as `BaseContext` without auditing every `ctx.Err()` in the
request path.

## Pattern closed

"Story-title oversell on partial graceful-shutdown" — closed by 12.14. Full `srv.Shutdown(30s)`,
`pool.Close()`, and ctx-aware `cleanupLoop` are all wired. Pattern kept in MEMORY as a watch-flag
for future "graceful X" stories.
